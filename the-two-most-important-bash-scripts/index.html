The two most important bash scripts

05 April 2015

If you want to save yourself time in the Mac OS Terminal, you will need to fill your bash profile with useful shortcuts. You get to choose how deep you will get into this game, which is to say, how easy your life in the Terminal will become. You'll probably start by copy-pasting useful shortcuts into your bash profile from source such as Josh Wyatt's list of keyboard shortcuts, then move onto creating your own aliases. Maybe you'll eventually start writing bash functions, too.

However, populating your bash profile is a tedious process of its own. Your bash profile is a dotfile, invisible to you as you browse the Finder window or type an ordinary "ls" in your terminal. After finding the bash profile and adding or modifying one of your aliases, you need to save the bash profile and restart the Terminal window. The repetitiveness of this process prevents many people from building up a fast Terminal workflow. In this post, I will tell you how to streamline that effort, making the process much easier for you.

Your bash profile resides at "/Users/steve/.bash_profile", and you can see it by navigating to your home folder (don't type the dollar sign; that's just there to indicate you're typing in the Terminal window)...

$ cd

...and then listing the files at this location.

$ ls

Wait, the bash profile's not there! Where is it? Try again, this time listing all the files:

$ ls -a

Those files whose names begin with a period? Those are dotfiles. Open up the bash profile with this command:

$ open .bash_profile

(Feel free to substitute a different text editor for 'open'). Now you will see a TextEdit window with your bash profile. Paste in these three commands:

alias subl="open -a Sublime\ Text\ 2"
alias prof="subl ~/.bash_profile"
alias reprof=". ~/.bash_profile"

Ok, so I lied about the "two" part. The two critical aliases are "prof" and "reprof", but as I have it here, "prof" relies on "subl". My "subl" command will work for users of the Sublime 2 text editor. If you use Sublime 3, substitute "3" for "2". If you don't use Sublime at all, you can adapt to the text editor of your choice, or just opt for this:

alias prof="open ~/.bash_profile"
alias reprof=". ~/.bash_profile"

See? There are only two.

Why are "prof" and "reprof" so useful? Just watch. Save and close your profile. Now, for the last time ever, restart your Terminal to put the new bash aliases into effect.

Now, think of some bash command you always type. If you use git, maybe you are always typing "git push origin master", and you wish you could just type "gpom". Well, first make sure "gpom" is not already taken:

 $ gpom
 >>> -bash: gpom: command not found

Looks like you're good to go, you don't need to worry about overwriting something important. Type 'prof', and you will see your bash profile open up. Add a new line to create your desired alias:

 alias gpom="git push origin master"

Sove the bash profile, return to the Terminal window, and type "reprof". Now your Terminal knows about "gpom" and you can use it right away. No need to restart.

This simple pair of commands can turn you into a true Terminal wizard. Whenever you're about to type a command that you've typed several times before, you can add a new alias or function without interrupting your workflow at all. Simply use "prof" and "reprof".

This blogpost was inspired by a talk given by Josh Wyatt, whose list of keyboard shortcuts is well worth reading. It has some nice starter aliases that you can copy into your bash profile, and also lists plenty of MacOS commands that helped ease my transition from Windows.

Aliases are fine, but bash functions are even more powerful. To see an example of a useful bash function, you can check out Chris Rinaldi's post about how to pull down a "solution" branch for several repos at a time.
